====================================================
DIAGRAMA ER — 7 TABLAS
====================================================

configuracion_empresa (singleton, id siempre = 1)
  PK id INT AUTO_INCREMENT
  nombre_empresa VARCHAR(100)
  cif VARCHAR(20) UNIQUE
  direccion TEXT
  email VARCHAR(100)
  telefono VARCHAR(20)
  logo_path VARCHAR(255)

usuarios
  PK id INT AUTO_INCREMENT
  username VARCHAR(50) UNIQUE
  password_hash VARCHAR(255)
  email VARCHAR(100) UNIQUE
  rol [ADMIN|ENCARGADO|EMPLEADO]
  activo BOOLEAN DEFAULT TRUE
  fecha_creacion TIMESTAMP
  reset_token VARCHAR(255) NULL       ← NUEVO
  reset_token_expiry DATETIME NULL    ← NUEVO
  IDX: username*, activo, reset_token*

empleados
  PK id INT AUTO_INCREMENT
  FK usuario_id → usuarios UNIQUE
  nombre / apellido1 / apellido2
  dni VARCHAR(20) UNIQUE
  nss VARCHAR(20) UNIQUE
  fecha_alta DATE
  jornada_semanal_horas INT
  dias_vacaciones_anuales INT
  dias_asuntos_propios_anuales INT
  pin_terminal CHAR(4) UNIQUE
  codigo_nfc VARCHAR(50) UNIQUE
  activo BOOLEAN
  IDX: pin_terminal*, dni, activo

fichajes
  PK id INT AUTO_INCREMENT
  FK empleado_id → empleados
  fecha DATE
  tipo [NORMAL|VAC|FESTIVO|BAJA...]
  hora_entrada DATETIME NULL
  hora_salida DATETIME NULL
  total_pausas_minutos INT
  jornada_efectiva_minutos INT (ceil)
  FK usuario_id → usuarios (auditoría)
  observaciones TEXT
  UNIQUE: (empleado_id, fecha) — 1 fichaje/día

pausas
  PK id INT AUTO_INCREMENT
  FK empleado_id → empleados
  fecha DATE
  hora_inicio DATETIME
  hora_fin DATETIME NULL (activa)
  duracion_minutos INT (floor)
  tipo_pausa [COMIDA|DESC|AUS_RET|OTROS]
  FK usuario_id → usuarios (auditoría)
  IDX: (empleado_id, fecha)*

planificacion_ausencias
  PK id INT AUTO_INCREMENT
  FK empleado_id → empleados *** NULL OK *** (NULL = festivo global)
  fecha DATE
  tipo_ausencia [FESTIVO|VAC|ASUNTO|PERM...]
  procesado BOOLEAN DEFAULT FALSE
  FK usuario_id → usuarios (auditoría)
  observaciones TEXT
  fecha_creacion TIMESTAMP
  UNIQUE: (empleado_id, fecha)

saldos_anuales
  PK id INT AUTO_INCREMENT
  FK empleado_id → empleados
  anio INT
  dias_trabajados INT
  dias_vacaciones_derecho_anio INT
  dias_vacaciones_pendientes_anterior INT
  dias_vacaciones_consumidos INT
  dias_vacaciones_disponibles INT
  dias_asuntos_propios_* INT (x4)
  saldo_horas DECIMAL(10,2)
  calculado_hasta_fecha DATE
  UNIQUE: (empleado_id, anio) — 1 registro/año

RF ADMIN — CERRADOS (RF-01 a RF-30 + auth)
====================================================

--- Configuración de empresa ---
RF-01  Registrar y editar los datos de la empresa (nombre, CIF, dirección, email, teléfono y logo)
RF-02  Consultar la configuración actual de la empresa

--- Usuarios ---
RF-03  Crear, editar y desactivar cuentas de usuario (username, email, rol y contraseña)

--- Empleados ---
RF-04  Crear, editar y desactivar fichas de empleado (nombre, apellidos, DNI, NSS, fecha de alta,
       jornada semanal, vacaciones anuales, asuntos propios, PIN de terminal y código NFC)
RF-05  Activar un empleado previamente desactivado
RF-06  Consultar el listado completo de empleados activos e inactivos
RF-07  Consultar el perfil completo de un empleado (datos personales, estado actual,
       jornada semanal y saldos anuales)
RF-08  Buscar empleados aplicando filtros opcionales por nombre, apellidos y/o estado (activo/inactivo)
RF-09  Consultar el estado actual de todos los empleados en tiempo real (dentro, fuera o en pausa)

--- Fichajes ---
RF-10  Registrar manualmente un fichaje para una fecha concreta para cualquier empleado
       (entrada, salida o tipo de jornada), con observaciones obligatorias
RF-11  Modificar un fichaje existente de cualquier empleado, con observaciones obligatorias
RF-12  Consultar el historial completo de fichajes de cualquier empleado con filtros por fecha
RF-13  Consultar los fichajes registrados en una fecha concreta para todos los empleados
RF-14  Consultar los fichajes incompletos (solo entrada o solo salida) de cualquier empleado o de todos

--- Pausas ---
RF-15  Registrar manualmente una pausa para cualquier empleado (tipo, hora inicio, hora fin),
       con observaciones obligatorias
RF-16  Modificar una pausa existente de cualquier empleado, con observaciones obligatorias
RF-17  Consultar el historial completo de pausas de cualquier empleado o de todos, con filtros por fecha

--- Ausencias ---
RF-18  Planificar ausencias para cualquier empleado (festivo, vacaciones, asunto, permiso u otro)
       con fecha y marcarlas como procesadas o pendientes
RF-19  Modificar una planificación de ausencia existente
RF-20  Eliminar una planificación de ausencia que aún no haya sido procesada (procesado = false)
RF-21  Consultar las ausencias de cualquier empleado o de todos, filtradas por período
       (pasadas y futuras)

--- Control y presencia ---
RF-22  Consultar el parte diario de presencia (quién ha fichado, a qué hora, quién falta
       y quién tiene ausencia planificada o justificada para ese día)
RF-23  Consultar los empleados que no han registrado fichaje en una fecha concreta

--- Informes ---
RF-24  Generar un informe de horas trabajadas por empleado en un período determinado
RF-25  Generar un informe del total de horas trabajadas por todos los empleados en un período determinado
RF-26  Consultar el estado de vacaciones y asuntos propios de todos los empleados
       (días asignados, consumidos, pendientes y pendientes de año anterior)
RF-27  Consultar y forzar el recálculo del saldo anual de cualquier empleado para un año concreto

--- Informes PDF firmables ---
RF-28  Generar un informe PDF de resumen mensual de un empleado para un mes y año concretos
       (fichajes, pausas, saldo de horas y estado de vacaciones y asuntos propios acumulados),
       para su impresión y firma
RF-29  Generar un informe PDF de cierre anual de un empleado (horas trabajadas, saldo de horas,
       vacaciones y asuntos propios consumidos y pendientes), para su impresión y firma
RF-30  Generar un informe PDF de vacaciones y asuntos propios de un empleado (días disfrutados,
       pendientes y pendientes de año anterior), para su impresión y firma

--- Autenticación ---
RF-AUTH-01  Autenticar a cualquier usuario en el sistema (login con username y contraseña)
RF-AUTH-02  Cerrar sesión (logout — invalida el JWT activo)
RF-AUTH-03  Solicitar recuperación de contraseña introduciendo el email registrado
RF-AUTH-04  Establecer una nueva contraseña mediante el enlace recibido por email
            (token de un solo uso con caducidad de 30 minutos)

====================================================
DECISIONES DE DISEÑO — API
====================================================

1. Registros inmutables
   Los fichajes y pausas NUNCA se eliminan.
   Los errores se corrigen con modificación + observaciones obligatorias.
   Cumplimiento RD-ley 8/2019.

2. Eliminación de ausencias
   Solo se pueden eliminar ausencias con procesado = false (aún no ocurridas).

3. Filtros opcionales y combinables
   Todos los endpoints GET aceptan parámetros opcionales combinables.

4. Empleados nunca se borran
   Un empleado desactivado permanece en el sistema para conservar su historial.

5. Informes imprimibles desde pantalla
   Los endpoints GET aceptan el parámetro ?formato=html.
   El backend devuelve HTML limpio (sin botones ni navegación).
   La app Android imprime ese HTML con PrintManager + WebView.

6. Informes de firma
   RF-28, RF-29 y RF-30 generan PDF con iText 7.
   El resto de informes usan ?formato=html.

7. Recuperación de contraseña
   Flujo: email → token temporal 30 min → enlace de reset → token invalidado.
   Requiere spring-boot-starter-mail + cuenta SMTP Gmail.
   Campos añadidos a tabla usuarios: reset_token VARCHAR(255) NULL
                                      reset_token_expiry DATETIME NULL

8. Autenticación por PIN (EMPLEADO)
   El EMPLEADO puede autenticarse con PIN de 4 dígitos desde el terminal,
   sin necesidad de username y contraseña.

9. El ADMIN nunca ficha como empleado (no tiene empleado asignado).
   El ENCARGADO sí ficha (tiene empleado asignado).

10. Pantalla standby
    La app muestra una pantalla de standby con la hora en tiempo real
    esperando que un empleado introduzca su PIN. Se documenta en Bloque 4.
