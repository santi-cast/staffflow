DETALLES TÉCNICOS CRÍTICOS DEL BLOQUE 3:
  E02 obtenerUsuarioAutenticado():
    Username extraído del SecurityContext (cargado por JwtAuthFilter).
    Devuelve UsuarioResponse vía método privado toUsuarioResponse().
  E03 cambiarPassword():
    passwordEncoder.matches(passwordActual, hash) para verificar.
    @Transactional. Si no coincide → IllegalArgumentException → 400.
    PasswordChangeRequest campos: passwordActual, passwordNueva.
  E04 solicitarRecuperacion():
    Respuesta 200 idéntica si email existe o no (RNF-S04).
    Token: UUID.randomUUID(). Expiry: now().plusMinutes(30).
    STUB: token en log.info(). @Transactional.
  E05 restablecerPassword():
    Busca por resetToken. Token inexistente o caducado → IllegalArgumentException.
    Tras uso: resetToken=null, resetTokenExpiry=null (1 solo uso, RNF-S04).
    @Transactional. PasswordResetRequest campos: token, passwordNueva.
  OpenApiConfig:
    Nombre "bearerAuth" idéntico en addSecurityItem() y addSecuritySchemes().
    SecurityRequirement global. Usuario pega token sin prefijo Bearer.
  EmpresaService E06-E07:
    E06: findById(1L) → IllegalStateException si no existe → 404.
    E07: findById(1L).orElse(new ConfiguracionEmpresa()). setId(1L) fuerza INSERT id=1.

    DataIntegrityViolationException si CIF duplicado → GlobalExceptionHandler → 400.
  GlobalExceptionHandler:
    Formato: { error, campo?, timestamp, path } con LinkedHashMap.
    IllegalArgumentException → 400 | IllegalStateException → 404
