====================================================
DECISIONES DE DISEÑO CERRADAS (24)
====================================================

1.  Fichajes/pausas inmutables — sin DELETE (RD-ley 8/2019, RNF-L01)
2.  Ausencias: DELETE solo si procesado=false
3.  Filtros GET opcionales y combinables
4.  Bajas lógicas: activo=false, nunca DELETE físico
5.  Informes HTML: ?formato=html → PrintManager+WebView Android
6.  PDF firmables: iText 7 (RF-38, RF-39, RF-40)
7.  Ausencias planificadas vs justificadas imprevistas (flujos distintos)
8.  Terminal PIN vs fichaje manual (ADMIN nunca ficha desde terminal)
9.  Recuperación contraseña: token opaco 30min, 1 uso (RNF-S04)
10. Acceso datos por rol: EMPLEADO → HTTP 403 si datos ajenos
11. Terminal standby: hora en tiempo real + PIN siempre visible
12. Saldo anual: desglose completo siempre (RF-35, RF-36, RF-53)
13. DIA_LIBRE_COMPENSATORIO: valor unificado (horas+festivo trabajado)
14. Visibilidad datos por rol (ADMIN/ENCARGADO=todos, EMPLEADO=suyos)
15. Umbrales rendimiento en P95
16. Bloqueo PIN por dispositivo, no por empleado (5 fallos → 30s)
17. Observabilidad: usuario_id auditoría BD + SLF4J/Logback
18. Sin refresh token v1.0 — JWT 8h cubre jornada
    (access+refresh descartado: complejidad sin beneficio práctico)
19. CategoriaEmpleado informativo (no afecta permisos)
20. Swagger UI como documentación oficial API
21. PIN=velocidad fichaje terminal compartido / JWT=acceso datos personales
22. jornada_semanal_horas(contractual) + jornada_diaria_minutos(cálculo)
23. Terminal P01 raíz pública sin Drawer. Por rol: ADMIN→P13|ENCARGADO→P17|EMPLEADO→P09
24. Single Activity + Nav Component + Fragments. JWT en DataStore. Drawer dinámico por rol.
