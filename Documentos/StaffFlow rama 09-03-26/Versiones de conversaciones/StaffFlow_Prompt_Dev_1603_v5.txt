    Obtener todos los empleados activos.
    Para cada empleado sin fichaje en LocalDate.now():

    Solo ADMIN puede corregir dias pasados (E23 con restriccion de fecha).

  TAREA B -- Procesado planificaciones pendientes:
    Buscar planificacion_ausencias con procesado=false
    y fecha <= LocalDate.now().plusDays(1).
    Para cada registro:
      Verificar si ya existe fichaje para ese empleado en esa fecha.
      Si no existe: crear fichaje con TipoFichaje correspondiente
        al TipoAusencia (mapeo enum a enum).
      Marcar procesado=true.
    Festivos globales (empleadoId=null): crear fichaje para
      todos los empleados activos de esa fecha.

  TAREA C -- Actualizacion saldos anuales:
    Ejecutar SIEMPRE despues de Tarea A y Tarea B.
    Para cada empleado activo: llamar a la logica de recalculo
    de SaldoService (misma logica que E40, sin restriccion de rol).
    E40 queda como herramienta de correccion manual exclusiva del ADMIN.

  RESTRICCION DE FECHA PARA ENCARGADO (implementar en los services):
    E22, E23, E27, E28, E30 y E31 deben validar:
      Si rol == ENCARGADO y fecha != LocalDate.now() -> HTTP 403.
      Si rol == ADMIN -> sin restriccion de fecha.
    Esto garantiza que el ENCARGADO solo puede actuar sobre el dia
    actual y no puede modificar dias ya cerrados por ProcesoCierreDiario.
    El ADMIN puede modificar cualquier dia sin restriccion.
