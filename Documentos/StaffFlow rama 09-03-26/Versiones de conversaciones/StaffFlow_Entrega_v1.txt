El proyecto se entregará dividido en dos bloques principales:

  Bloque 1: Backend (API REST con base de datos H2 embebida)
  Bloque 2: Aplicación Android (terminal de fichaje y consulta)

El proyecto se ha preparado para que pueda ejecutarse directamente sin
necesidad de configuraciones complejas, facilitando así su evaluación.

La aplicación backend utiliza H2 Database embebida, por lo que no es
necesario instalar ni configurar un servidor de base de datos externo.

====================================================
BLOQUE 1 — BACKEND (API REST con base de datos H2 embebida)
====================================================

El backend del sistema está desarrollado mediante Spring Boot y
proporciona una API REST que gestiona la lógica de negocio del sistema.

Se ofrecen dos formas de ejecutar el backend:

OPCIÓN A — Ejecución desde entorno de desarrollo

  El backend puede ejecutarse directamente desde el proyecto utilizando
  un entorno de desarrollo como IntelliJ IDEA.

  Requisitos mínimos:
    Java    21
    Maven   3.9

OPCIÓN B — Ejecución mediante archivo ejecutable (.jar)

  El backend también se entregará como un archivo ejecutable (.jar)
  generado por Spring Boot.

  Comando: java -jar staffflow.jar

  Requisitos mínimos:
    Java    21

====================================================
BLOQUE 2 — APLICACIÓN ANDROID (terminal de fichaje y consulta)
====================================================

La aplicación Android está diseñada para ejecutarse en una tablet que
actúa como terminal de fichaje dentro de la empresa.

Desde este terminal los empleados pueden registrar su entrada y salida,
gestionar pausas y consultar información relacionada con su jornada
laboral.

Dependiendo del rol del usuario autenticado, la aplicación permite
acceder a diferentes funcionalidades. Los usuarios con rol encargado
pueden utilizar la aplicación tanto para registrar su propia jornada
como para acceder a funciones de supervisión y gestión.

La aplicación se comunica con la API REST del sistema para realizar
todas las operaciones.

OPCIÓN A — Ejecución desde Android Studio

  Requisitos mínimos:
    Android Studio compatible con Kotlin y AGP 9.0.1

OPCIÓN B — Instalación mediante archivo APK

  Se incluirá el archivo APK compilado de la aplicación.

====================================================
NOTAS
====================================================

- Docker descartado del plan de entrega (no forma parte del stack
  del proyecto).
- El perfil activo para la entrega evaluable es el perfil dev (H2),
  no el perfil mysql, para eliminar dependencias externas al evaluar.
