1. OBJETIVO
-----------
Verificar que los endpoints implementados en el Bloque 5 responden correctamente
a peticiones HTTP reales contra la base de datos MySQL en entorno local, confirmando
que la logica de negocio, la persistencia y los codigos de respuesta HTTP se ajustan
al diseno definido en Endpoints_v3 y BaseDatos_v4.

Los endpoints verificados cubren tres modulos funcionales:

  - Modulo de Fichajes: E22 a E26
  - Modulo de Pausas:   E27 a E29
  - Modulo de Terminal: E48 a E51

5. MODULO DE TERMINAL (E48-E51)
---------------------------------
Los endpoints del terminal son publicos (sin JWT). La autenticacion se realiza
exclusivamente mediante PIN de 4 digitos, segun la decision de diseno n.21.

5.1 E48 - POST /api/v1/terminal/entrada
Descripcion: Registrar la entrada de un empleado desde el terminal fisico por PIN.
Prueba 1 - PIN de Ana (1111), que ya tenia fichaje registrado hoy:
  Body: { "pin": "1111", "dispositivoId": "terminal-01" }
  Resultado: 409 Conflict - "El empleado ya tiene registrada la entrada hoy 2026-03-13"
  Verificacion: el sistema detecto correctamente el intento duplicado.

Prueba 2 - PIN de Laura (3333), sin fichaje previo:
  Body: { "pin": "3333", "dispositivoId": "terminal-01" }
  Resultado: 200 OK
  Respuesta: { nombre: "Laura", horaEntrada: "2026-03-13T15:24:05.300", mensaje: "Entrada registrada" }
  Verificacion: fichaje creado en BD con tipo=NORMAL, horaSalida=null,
    totalPausasMinutos=0, usuario=terminal_service.

Incidencia resuelta durante las pruebas:
  Al registrar la entrada de Laura se obtuvo 400 Bad Request con el mensaje
  "Column 'tipo' cannot be null" en el log de IntelliJ. Se identifico que
  TerminalService.registrarEntrada() construia la entidad Fichaje sin asignar
  el campo tipo. Se corrigio anadiendo fichaje.setTipo(TipoFichaje.NORMAL)
  antes del save(). Esta correccion quedo registrada como desviacion D-022.
  Adicionalmente, el empleado Laura tenia activo=0 en la base de datos, lo que
  provocaba un 400 previo. Se actualizo a activo=1 mediante UPDATE directo
  para continuar con las pruebas.

5.2 E49 - POST /api/v1/terminal/salida
Descripcion: Registrar la salida y calcular la jornada efectiva.
Prueba 1 - PIN de Ana (1111), que ya tenia salida registrada:
  Resultado: 409 Conflict - "La salida ya esta registrada para hoy"
  Verificacion: el sistema impidio registrar una segunda salida.

Prueba 2 - PIN de Carlos (2222), con entrada activa:
  Body: { "pin": "2222", "dispositivoId": "terminal-01" }
  Resultado: 200 OK
  Respuesta: { nombre: "Carlos", horaSalida: "2026-03-13T15:18:18", jornadaEfectivaMinutos: 318, mensaje: "Salida registrada" }
  Verificacion: jornadaEfectivaMinutos=318 calculado correctamente con
    Math.ceil(minutosBrutos - totalPausasMinutos). El fichaje quedo cerrado en BD.

5.3 E50 - POST /api/v1/terminal/pausa/iniciar
Descripcion: Iniciar una pausa desde el terminal.
Prueba - PIN de Laura (3333), con entrada activa y sin pausa en curso:
  Body: { "pin": "3333", "tipoPausa": "COMIDA", "dispositivoId": "terminal-01" }
  Resultado: 200 OK
  Respuesta: { nombre: "Laura", horaInicioPausa: "2026-03-13T15:24:26", duracionPausaMinutos: null, mensaje: "Pausa iniciada" }
  Verificacion: pausa creada en BD con horaFin=null (pausa activa en curso)
    y tipoPausa=COMIDA.

5.4 E51 - POST /api/v1/terminal/pausa/finalizar
Descripcion: Finalizar la pausa activa.
Prueba - PIN de Laura (3333), con pausa COMIDA activa:
  Body: { "pin": "3333", "dispositivoId": "terminal-01" }
  Resultado: 200 OK
  Respuesta: { nombre: "Laura", horaInicioPausa: null, duracionPausaMinutos: 0, mensaje: "Pausa finalizada" }
  Verificacion: horaFin registrada en BD. duracionPausaMinutos=0 es correcto
    dado que la pausa duro menos de 1 minuto en el entorno de pruebas
    (Math.floor redondea a la baja). El campo totalPausasMinutos del fichaje
    de Laura no se incremento al ser 0 minutos, comportamiento conforme al
    diseno (COMIDA descuenta de jornada, pero 0 minutos no tiene efecto).

7. INCIDENCIAS DETECTADAS Y RESUELTAS
---------------------------------------

D-022 - Campo tipo null en TerminalService.registrarEntrada()
  Descripcion: Al intentar registrar la entrada de Laura desde el terminal,
    el servidor devolvio 400 Bad Request. El log de IntelliJ mostro la
    excepcion "Column 'tipo' cannot be null" al ejecutar el INSERT en fichajes.
  Causa raiz: TerminalService.registrarEntrada() construia la entidad Fichaje
    asignando todos los campos excepto tipo, que es NOT NULL en el DDL.
  Correccion aplicada: se annadio la linea
    fichaje.setTipo(TipoFichaje.NORMAL)
    antes de fichajeRepository.save(fichaje). Todas las entradas desde terminal
    fisico son jornadas ordinarias (NORMAL). Los tipos especiales los gestiona
    el encargado desde la aplicacion movil.
  Impacto: ninguno en el diseno. Correccion de implementacion menor.

Dato de prueba corregido - Laura activo=0
  Descripcion: el empleado Laura Fernandez (id=3) tenia el campo activo=0
    en la base de datos de desarrollo, lo que provocaba un 400 Bad Request
    al intentar fichar su entrada (empleado de baja no puede fichar, diseno n.4).
  Accion tomada: UPDATE SET activo=1 directamente en MySQL para habilitar
    el empleado en el entorno de pruebas.
  Impacto: ninguno en el codigo. Dato incorrecto en BD de desarrollo.


8. OBSERVACIONES
-----------------
- El sistema de bloqueo por dispositivo (RNF-S05) implementado en TerminalService
  mediante ConcurrentHashMap no fue probado explicitamente en este bloque.
  Su alcance es interno al servicio y queda fuera del scope de verificacion
  funcional de los endpoints.

- El encoding de la respuesta en PowerShell muestra caracteres erroneos en
  el campo "mensaje" (por ejemplo "â" en lugar del simbolo de verificacion).
  Esto es un problema de visualizacion del cliente, no del backend. El servidor
  devuelve UTF-8 correcto confirmado en el log de IntelliJ.

- La duracion de 0 minutos en la pausa de Laura (E51) es un resultado esperado
  en entorno de pruebas donde inicio y fin se ejecutan en segundos consecutivos.
  En produccion la duracion reflejaria el tiempo real de la pausa.
