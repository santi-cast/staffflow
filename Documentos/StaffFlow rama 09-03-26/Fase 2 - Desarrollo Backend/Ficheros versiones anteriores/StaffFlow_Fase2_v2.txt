TAREA 3: Implementar GlobalExceptionHandler
  @RestControllerAdvice — centraliza conversión de excepciones en respuestas HTTP.
  Formato contrato: { error, campo?, timestamp, path } con LinkedHashMap (orden garantizado).
  Handlers:
    MethodArgumentNotValidException → 400 con campo concreto
    IllegalArgumentException        → 400 (contraseña incorrecta, token inválido)
    DataIntegrityViolationException → 400 (UNIQUE violada en BD)
    BadCredentialsException         → 401 (login fallido)
    AccessDeniedException           → 403 (rol insuficiente)

  devolvían 404 en lugar de 409.
  Solución:
    - ConflictException.java creada en paquete exception (RuntimeException custom).
    - GlobalExceptionHandler.java: handler @ExceptionHandler(ConflictException.class) → HTTP 409.
    - UsuarioService.java: throws username/email duplicado → ConflictException.
    - EmpleadoService.java: throws DNI/NSS/PIN/NFC duplicado y reactivar() ya activo → ConflictException.
  Resultado: IllegalStateException queda exclusivamente para "recurso no encontrado" (404).
             ConflictException gestiona exclusivamente conflictos de unicidad (409).
