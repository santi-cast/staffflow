Este fichero documenta la implementacion de MainActivity.kt y los
dos singletons de soporte creados para gestionar la sesion y los
eventos de autenticacion.

====================================================
1. CONTEXTO
====================================================

MainActivity es la unica Activity de la app (Single Activity pattern).
Todas las pantallas son Fragments alojados en el NavHostFragment.
Esta tarea implementa el esqueleto de navegacion, el Drawer dinamico
por rol y la gestion global de sesion expirada.

====================================================
3. SessionManager.kt
====================================================

Wrapper de Preferences DataStore con patron Singleton.
Almacena 4 valores tras el login exitoso (E01):
  token       (String)  <- JWT. Expira en 12h (D-024).
  rol         (String)  <- nombre del enum Rol (ADMIN/ENCARGADO/EMPLEADO)
  username    (String)
  empleadoId  (Long?)   <- null si el rol es ADMIN

Metodos publicos:
  saveSession(token, rol, username, empleadoId)
  clear()       <- borra todo. Llamar al cerrar sesion o en 401.
  getToken()    <- String? (suspend)
  getRol()      <- Rol?   (suspend)
  getUsername() <- String? (suspend)
  getEmpleadoId() <- Long? (suspend)

====================================================
4. AuthEventBus.kt
====================================================

Singleton con MutableSharedFlow<AuthEvent>(extraBufferCapacity=1).
Permite que AuthInterceptor (hilo de red) notifique a MainActivity
(hilo principal) sin dependencia directa entre ellos.

  AuthEvent.SessionExpired <- emitido por AuthInterceptor al detectar
                              HTTP 401 en un endpoint autenticado.

post(event) usa tryEmit: no suspende, seguro desde un hilo de red.
MainActivity colecciona el flow en lifecycleScope y llama a
handleSessionExpired().

====================================================
5. NetworkModule.kt -- CAMBIOS
====================================================

Anadido AuthInterceptor antes del LoggingInterceptor:
  - Lee NetworkModule.authToken (@Volatile var).
  - Si no es null, adjunta header Authorization: Bearer {token}.
  - Si la respuesta es HTTP 401, llama a AuthEventBus.post(SessionExpired).

NetworkModule.authToken se establece en:
  - checkExistingSession() de MainActivity (arranque con sesion valida).
  - LoginFragment tras login exitoso (Tarea del Bloque 2).
  - handleSessionExpired() / showLogoutDialog() lo pone a null.

====================================================
6. MainActivity.kt -- IMPLEMENTACION
====================================================

Estructura de metodos:

  onCreate()
    setSupportActionBar(toolbar)
    navController <- NavHostFragment.navController
    appBarConfiguration <- topLevelDestinations + drawerLayout
    setupActionBarWithNavController()
    setupDrawerItemListener()
    setupDestinationListener()
    collectAuthEvents()
    checkExistingSession()

  refreshDrawerMenu()
    Lee rol y username del DataStore.
    Actualiza tvHeaderUsername y tvHeaderRol en la cabecera.
    Muestra/oculta grupos segun rol:
      EMPLEADO  -> group_empleado + group_ajustes
      ENCARGADO -> group_empleado + group_encargado + group_ajustes
      ADMIN     -> todos los grupos

  hideDrawerMenu()
    Oculta todos los grupos. Llamar al cerrar sesion o en 401.

  navigateToInitialDestination(rol)
    Navega al destino inicial por rol limpiando el back stack
    con popUpTo(terminalFragment, inclusive=true):
      ADMIN     -> empleadosFragment  (P13)
      ENCARGADO -> parteDiarioFragment (P17)
      EMPLEADO  -> miSaldoFragment    (P09)
    Llamado desde LoginFragment tras login exitoso.

  setupDestinationListener()
    Escucha cambios de destino del NavController.
    Zona publica (terminalFragment, loginFragment, recoveryFragment,
    resetPasswordFragment):
      drawerLayout.LOCK_MODE_LOCKED_CLOSED + toolbar.navigationIcon=null
    Zona autenticada:
      drawerLayout.LOCK_MODE_UNLOCKED

  setupDrawerItemListener()
    nav_cerrar_sesion -> showLogoutDialog() (gestion manual).
    Resto -> NavigationUI.onNavDestinationSelected() (automatico).

  checkExistingSession()
    Lee token del DataStore al arrancar.
    Si existe: NetworkModule.authToken = token -> refreshDrawerMenu()
               -> navigateToInitialDestination(rol)
    Si no existe: permanece en terminalFragment (start destination).

  handleSessionExpired()
    sessionManager.clear() + NetworkModule.authToken = null
    hideDrawerMenu() -> navega a loginFragment (pop hasta root)
    Snackbar: "La sesion ha caducado"

  showLogoutDialog()
    MaterialAlertDialogBuilder (Decision 26).
    Boton destructivo: setNegativeButton ("Cerrar sesion").
    Al confirmar: clear() + hideDrawerMenu() -> loginFragment.

  onSupportNavigateUp()
    NavigationUI.navigateUp(navController, appBarConfiguration)

====================================================
7. DECISIONES DE IMPLEMENTACION
====================================================

Decision 1 -- findFragmentById en lugar de Activity.findNavController():
  FragmentContainerView requiere obtener el NavController via
  supportFragmentManager.findFragmentById() cast a NavHostFragment.
  El metodo Activity.findNavController() puede fallar en algunos
  dispositivos con FragmentContainerView.

Decision 2 -- NavOptions.popUpTo en navigateToInitialDestination:
  Se limpia el back stack hasta terminalFragment (inclusive) al
  navegar post-login. Evita que el boton Atras del sistema devuelva
  al terminal o al login una vez autenticado.

Decision 3 -- @Volatile var authToken en NetworkModule:
  Permite que LoginFragment escriba el token y el AuthInterceptor
  lo lea sin runBlocking ni DataStore en el hilo de red.
  Es la solucion mas simple para DI manual sin Hilt.

Decision 4 -- topLevelDestinations incluye los tres destinos iniciales:
  Los tres destinos de entrada por rol (P09, P13, P17) son
  top-level: muestran hamburguesa en lugar de boton Atras.
  Esto es correcto porque el usuario no tiene a donde "volver"
  desde esos fragments.
