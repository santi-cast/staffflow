Este fichero documenta la creacion del layout principal de la app
y los recursos asociados al Navigation Drawer (menu, header y strings).

====================================================
1. CONTEXTO
====================================================

La app sigue el patron Single Activity + Navigation Component.
MainActivity es la unica Activity; todos los fragments se alojan
en el NavHostFragment definido en este layout. El Drawer es el
mecanismo de navegacion principal para la zona autenticada.

El layout existente (generado por Android Studio) era un
ConstraintLayout con un TextView "Hello World!" y no servia
como base para la estructura requerida.

====================================================
2. FICHEROS CREADOS O MODIFICADOS
====================================================

Modificados:
  res/layout/activity_main.xml   <- reemplazado por completo
  res/values/strings.xml         <- anadidas strings del Drawer

Creados:
  res/menu/drawer_menu.xml       <- menu del Navigation Drawer
  res/layout/nav_header_main.xml <- cabecera del Drawer

====================================================
3. ESTRUCTURA DE activity_main.xml
====================================================

DrawerLayout (raiz, id: drawerLayout)
  |
  +-- CoordinatorLayout (contenido principal)
  |     |
  |     +-- AppBarLayout (id: appBarLayout)
  |     |     |
  |     |     +-- MaterialToolbar (id: toolbar)
  |     |
  |     +-- FragmentContainerView (id: navHostFragment)
  |           NavHostFragment apuntando a @navigation/nav_graph
  |           app:defaultNavHost="true"
  |           layout_behavior: appbar_scrolling_view_behavior
  |
  +-- NavigationView (id: navigationView, gravity: start)
        headerLayout: nav_header_main
        menu: drawer_menu

Tema: Theme.Staffflowandroid hereda de
  Theme.Material3.DayNight.NoActionBar. La Toolbar es el
  ActionBar de la app; se configura en MainActivity con
  setSupportActionBar(binding.toolbar).

====================================================
4. drawer_menu.xml -- GRUPOS Y VISIBILIDAD
====================================================

El menu tiene 4 grupos que MainActivity muestra u oculta segun
el rol leido del DataStore tras el login:

  group_empleado  (checkableBehavior="single")
    Visible: EMPLEADO, ENCARGADO, ADMIN
    Items: miHoyFragment (P12) | misFichajesFragment (P10) |
           misAusenciasFragment (P11) | miSaldoFragment (P09) |
           miPerfilFragment (P08)

  group_encargado (checkableBehavior="single")
    Visible: ENCARGADO, ADMIN
    Items: parteDiarioFragment (P17) | empleadosFragment (P13) |
           fichajesFragment (P19) | ausenciasFragment (P23) |
           saldosGlobalesFragment (P27) | informesFragment (P28)

  group_admin (checkableBehavior="single")
    Visible: ADMIN
    Items: usuariosFragment (P31) | empresaFragment (P34)

  group_ajustes
    Visible: todos los roles autenticados
    Items: cambiarPasswordFragment (P04) | nav_cerrar_sesion

Los IDs de los items de navegacion (excepto nav_cerrar_sesion)
coinciden con los IDs de destino en nav_graph.xml. Esto permite
que NavigationUI.setupWithNavController() gestione la navegacion
automaticamente al pulsar cada item del Drawer.

nav_cerrar_sesion no tiene destino en el nav_graph: se gestiona
manualmente en MainActivity con un AlertDialog de confirmacion
(Decision 26 -- MaterialAlertDialogBuilder).

====================================================
5. nav_header_main.xml -- CABECERA DEL DRAWER
====================================================

LinearLayout vertical con fondo colorPrimary:
  tvHeaderAppName  <- nombre de la app ("StaffFlow")
  tvHeaderUsername <- username del usuario autenticado
  tvHeaderRol      <- rol del usuario (ADMIN/ENCARGADO/EMPLEADO)

Los campos tvHeaderUsername y tvHeaderRol se rellenan en
MainActivity tras leer el DataStore. Los colores usan
colorOnPrimary para garantizar contraste con el fondo.

====================================================
6. BLOQUEO DEL DRAWER EN ZONA PUBLICA
====================================================

Los fragments de la zona publica (P01 Terminal, P02 Login,
P03 Recovery, P05 Reset) no deben mostrar el Drawer.
MainActivity escucha los cambios de destino del NavController
y aplica:

  Zona publica:
    drawerLayout.setDrawerLockMode(DrawerLayout.LOCK_MODE_LOCKED_CLOSED)
    toolbar.navigationIcon = null

  Zona autenticada:
    drawerLayout.setDrawerLockMode(DrawerLayout.LOCK_MODE_UNLOCKED)
    ActionBarDrawerToggle activo

Esto se implementa en Tarea 5 (MainActivity.kt).

====================================================
7. DECISIONES DE IMPLEMENTACION
====================================================

Decision 1 -- FragmentContainerView en lugar de fragment tag:
  La documentacion oficial de Navigation Component recomienda
  FragmentContainerView sobre el tag <fragment> deprecado.
  Evita problemas con el ciclo de vida del fragment en
  transacciones manuales.

Decision 2 -- IDs de menu iguales a IDs de destino en nav_graph:
  Permite usar NavigationUI.setupWithNavController() sin codigo
  adicional para la mayoria de las navegaciones del Drawer.
  Solo nav_cerrar_sesion requiere gestion manual.

Decision 3 -- Grupos en lugar de menus separados por rol:
  Un unico drawer_menu.xml con grupos visibles/ocultos por rol
  es mas sencillo de mantener que tres ficheros de menu distintos.
  La visibilidad se controla con menu.setGroupVisible() en
  MainActivity al leer el rol del DataStore.
