Este fichero contiene todo el contexto del backend necesario para
implementar la aplicacion Android de StaffFlow sin necesidad de
adjuntar ningun fichero Java del backend. Incluye todos los DTOs,
enums, endpoints, data classes Kotlin, y decisiones de arquitectura.

====================================================
1. STACK ANDROID
====================================================

Lenguaje:          Kotlin
AGP:               9.0.1
Arquitectura:      Single Activity + Navigation Component
Networking:        Retrofit 2.9.0 + OkHttp 4.12.0 + logging-interceptor
Persistencia:      Preferences DataStore 1.1.1 (NO SharedPreferences)
Async:             Coroutines 1.8.1 + StateFlow/LiveData
Lifecycle:         ViewModel + Runtime 2.8.0
Navigation:        Navigation Component 2.8.0
KSP:               2.1.0-1.0.29 (NO kapt -- incompatible con AGP 9.x)
DI:                Manual con object singleton (SIN Hilt -- incompatible AGP 9.x)
UI:                Material Design 3

BASE URL emulador: http://10.0.2.2:8080/api/v1/
BASE URL prod:     configurar en BuildConfig

NetworkModule.kt (ya existe en el proyecto):
  OkHttpClient con HttpLoggingInterceptor (Level.BODY)
  Timeouts: 30s connect/read/write
  Retrofit con GsonConverterFactory
  object singleton

3. ENUMS KOTLIN
====================================================

enum class Rol { ADMIN, ENCARGADO, EMPLEADO }

enum class TipoFichaje {
    NORMAL, FESTIVO_NACIONAL, FESTIVO_LOCAL,
    VACACIONES, ASUNTO_PROPIO, PERMISO_RETRIBUIDO,
    BAJA_MEDICA, DIA_LIBRE_COMPENSATORIO, AUSENCIA_INJUSTIFICADA
}

enum class TipoAusencia {
    FESTIVO_NACIONAL, FESTIVO_LOCAL, VACACIONES,
    ASUNTO_PROPIO, PERMISO_RETRIBUIDO, DIA_LIBRE_COMPENSATORIO
}

enum class TipoPausa { COMIDA, DESCANSO, AUSENCIA_RETRIBUIDA, OTROS }

enum class CategoriaEmpleado {
    OPERARIO, ADMINISTRATIVO, TECNICO, ENCARGADO, OTRO
}

enum class EstadoPresencia {
    JORNADA_INICIADA, EN_PAUSA, JORNADA_COMPLETADA,
    AUSENCIA_REGISTRADA, AUSENCIA_PLANIFICADA, SIN_JUSTIFICAR
}

Colores por EstadoPresencia en Android:
  JORNADA_INICIADA    -> verde
  EN_PAUSA            -> naranja
  JORNADA_COMPLETADA  -> verde (mas oscuro)
  AUSENCIA_REGISTRADA -> azul
  AUSENCIA_PLANIFICADA-> azul (mas claro)
  SIN_JUSTIFICAR      -> rojo

7. PATRON DE MANEJO DE ERRORES EN ANDROID
====================================================

Todos los errores del backend devuelven ErrorResponse:
  {"status": 404, "error": "Not Found", "mensaje": "...", "timestamp": "..."}

Codigos relevantes para Android:
  400 -> datos invalidos, mostrar mensaje en campo concreto
  401 -> credenciales invalidas o token expirado
  403 -> rol insuficiente (no deberia ocurrir si la UI oculta bien las opciones)
  404 -> recurso no encontrado
  409 -> conflicto (duplicado, procesado=true, etc.)
  423 -> dispositivo bloqueado (terminal: 5 intentos fallidos)
  500 -> error interno del servidor

Flujo sesion expirada (401 en endpoint autenticado):
  DataStore.clear() -> hideDrawer() -> navigate(P02) ->
  snackbar("La sesion ha caducado")

11. MANEJO DE PDFs EN ANDROID
====================================================

Los endpoints E45, E46 y E47 devuelven byte[] con Content-Type application/pdf.
Para abrir el PDF en Android:

1. Guardar byte[] en fichero temporal:
   val file = File(context.cacheDir, nombreFichero)
   file.writeBytes(pdfBytes)

2. Obtener URI con FileProvider:
   val uri = FileProvider.getUriForFile(context, "${context.packageName}.provider", file)

3. Abrir con Intent:
   val intent = Intent(Intent.ACTION_VIEW).apply {
       setDataAndType(uri, "application/pdf")
       addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION)
   }
   startActivity(intent)

Requiere FileProvider en AndroidManifest.xml y res/xml/file_paths.xml.
